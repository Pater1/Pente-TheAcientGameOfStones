Parameters

exactTypeMatch
Type: System..::.Boolean
The value indicating whether exact type match should be performed. If it equals the type match is exact. That is, type B matches only type B and does not match any other type. If it equals , types are matched based on their type compatibility. That is, type B matches not only itself, but also any other type compatible with it. E.g., type D, derived from B. That effectively allows factories to be registered not only for types, but for interfaces as well. In case there is no exact match for a type, a factory for the 'closest' compatible type is chosen. That type is determined by the following algorithm:
  1. The map is searched for all base type of the type in question (including Object) in the order of the inheritance chain. If the map contains an entry for a certain base type, the corresponding ObjectGraphFactory is returned, if not the next step is taken.
  2. The map is searched for types of interfaces implemented by the type in question. If the map contains an entry for a certain interface the corresponding ObjectGraphFactory is returned. If no entries are found the KeyNotFoundException is thrown. If more than one entry is found the ArgumentException is thrown.
